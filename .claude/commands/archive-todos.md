# Archive Completed TODOs

你是一個 TODO 管理助手。請執行以下步驟來整理和封存已完成的任務：

## 步驟 1: 讀取並分析 TODO.md

1. 讀取專案根目錄的 `TODO.md` 檔案
2. 識別所有標記為已完成的任務（通常標記為 `[x]`、`✓`、`✅`、`DONE`、`完成` 等）
3. 列出所有已完成的任務，並簡要說明每個任務的內容

## 步驟 2: 驗證任務完成狀態

對於每個標記為已完成的任務：

1. **檢查相關檔案**: 如果任務提到特定的檔案或功能，檢查這些檔案是否存在
2. **檢查 Git 提交**: 查看最近的 git log，確認是否有相關的提交記錄
3. **搜尋相關程式碼**: 使用 Grep 工具搜尋任務提到的功能或組件
4. **驗證實作**: 快速檢查關鍵功能是否已實作

對於每個任務，給出驗證結果：
- ✅ **確認完成**: 有明確證據顯示任務已完成
- ⚠️ **部分完成**: 任務部分完成但可能需要後續工作
- ❌ **未完成**: 沒有證據顯示任務已完成（標記錯誤）

## 步驟 3: 創建封存檔案

1. 檢查是否存在 `doc/archive/``TODO_ARCHIVE.md` 檔案
2. 如果不存在，創建新的封存檔案，包含標題和說明
3. 在封存檔案中添加一個新的封存區塊，包含：
   - 封存日期和時間
   - 封存的任務數量
   - 每個已完成任務的詳細資訊（原始任務描述、完成時間推測）

封存格式範例：
```markdown
## 封存 - 2025-01-15

封存了 5 個已完成的任務

### ✅ 任務標題
- **完成時間**: 推測 2025-01-14
- **描述**: 任務的詳細描述
- **驗證**: 確認已完成（檔案存在、功能已實作）

---
```

## 步驟 4: 更新 TODO.md

1. 從 `TODO.md` 中移除所有已驗證完成的任務
2. 保留部分完成或未完成的任務（並添加備註說明狀態）
3. 重新組織剩餘的任務，保持清晰的結構
4. 在檔案頂部添加最後更新時間

## 步驟 5: 生成摘要報告

提供一份清晰的摘要報告，包含：

```
📊 TODO 整理摘要報告

已封存任務: X 個
- ✅ 確認完成: X 個
- ⚠️ 部分完成: X 個 (保留在 TODO.md 中)
- ❌ 標記錯誤: X 個 (保留在 TODO.md 中)

剩餘待辦任務: X 個

📁 檔案更新:
- TODO.md: 已更新並簡化
- TODO_ARCHIVE.md: 已添加 X 個任務記錄

💡 建議:
- [列出任何需要注意的事項]
```

## 重要提示

- **謹慎處理**: 只封存有明確證據顯示已完成的任務
- **保留上下文**: 在封存時保留足夠的上下文資訊，方便日後查詢
- **保持格式**: 確保 TODO.md 和 TODO_ARCHIVE.md 都保持良好的 Markdown 格式
- **不刪除重要資訊**: 如果任務包含重要的技術決策或設計文件連結，在封存時保留這些資訊
- **檔案位置**: 新建立的檔案請勿存放於根目錄


現在請開始執行整理流程。
