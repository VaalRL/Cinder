# 02｜任務執行：計畫與多步驟協作

## 何時使用

- 任務 **>= 3 步**
- 操作可能 **> 30 秒**
- 任務同時跨越 `packages/core/`、`apps/desktop/`、`relay/` 或測試層
- 需要先做 PoC、驗證連線延遲、Nostr 事件契約或 API 邊界

## 作法

1. **先拆任務**
   - 為每個子任務定義輸入、輸出、完成條件、風險與回復方式。

2. **先做不確定性最高的驗證**
   - 優先處理會阻塞整體設計的部分，例如 relay 連線、事件簽章/驗證、WebRTC 信令契約。

3. **設定檢查點**
   - 在模組邊界改動、資料流定案、測試基線建立後做檢查點。

4. **每一子任務都要可驗收**
   - 以測試、PoC 結果或文件更新確認該步驟已完成，而不是只靠口頭推論。

## 產出

- 清楚的執行計畫與優先順序
- 驗證結果或自動化測試
- 若有設計收斂，更新 `ARCHITECTURE.md`、`README.md` 或 `AGENTS.md`
