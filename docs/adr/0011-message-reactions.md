# 0011. 訊息回應（Reactions，M6）

- 狀態：已接受
- 日期：2026-06-30
- 相關文件：docs/adr/0010（M6 路線圖）；ARCHITECTURE.md §5

## 背景與問題

M6 首個功能：對訊息按 emoji 回應。需在不破壞隱私（隱藏誰回應誰）與既有 1:1 加密通道下實作。

## 決策

- **協定**：NIP-25 回應為 rumor kind 7（`e` tag 指向目標訊息 id、content = emoji），包進 **kind 1059 Gift Wrap**，與私訊走同一條加密通道；收件端以 `rumor.kind` 分流（14=訊息、7=回應）。
- **UI（v1）**：每則訊息顯示「不重複的 emoji 徽章」（不顯示計數與回應者），訊息旁提供加回應選單。
- **持久化**：回應本機儲存並於重載回放；以回應事件 id（`seenMsg` + storage）去重，避免 relay 重送造成重複。

## 後果

- 正面：沿用私訊通道與 NIP-17/59 隱私機制；核心 `wrapReaction`/`reactionTarget` 純函式可測，端到端經真實 relay 驗證。
- 負面 / 未來：v1 不顯示計數/回應者、不支援移除回應；同一 emoji 由多人回應會合併為單一徽章。可於後續版本擴充。
