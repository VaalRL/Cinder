# 0102. 圖片跨 session：持久化縮圖＋從 savedPath 讀回原圖＋原檔搬走可重新指定（收斂 ADR-0023）

- 狀態：已接受（已實作）
- 日期：2026-07-13
- 相關文件：ADR-0023（媒體相簿，其跨 session 方案已作廢）、0020（SQLite，從未實作、已被 0054 取代）、
  0093（檔案接收：**App 不保存位元組**）、0054（加密儲存基質）；
  `packages/engine/src/storage/types.ts`、`apps/desktop/src/ui/thumbnail.ts`、`apps/mobile/src/native/files.ts`

## 背景與問題

ADR-0023 的相簿是**零持久化**設計：從對話訊息**即時衍生**——篩出「帶 `url` 的 `image/*` 檔案訊息」。
它自己也寫明限制：

> v1 範圍為「單一對話、**目前 session 收發過且仍持有 blob 的圖片**」；跨 session 需搭配**原生持久化
> （B4 SQLite）落地媒體位元組**後重建（尚未接線）。

**它依賴的那條路已經斷了兩次：**
1. **B4（ADR-0020 SQLite/SQLCipher）從未實作**——`rusqlite` 從沒被建置，已被 ADR-0054 的加密 blob 取代。
2. **ADR-0093 直接否決了這個方向**——使用者裁示「**不用在上面保存檔案本身**」，改為收檔另存到使用者
   選定路徑，App 只留 metadata＋`savedPath`。

所以這不是「有設計但沒做」，而是**原設計已被後來的決定取代，新設計還沒有**。

實際症狀：**重載後相簿是空的，聊天裡的圖片退化成灰色檔案卡。**

## 考量的選項

- **A. 只從 `savedPath` 讀回原圖**：真正零位元組儲存。但只限 Tauri、只對**收到**的檔案有效
  （送出端沒記路徑），且使用者把檔案搬走就失效。
- **B. 只存縮圖**：全平台可靠。但**確實是在存位元組**（衍生的預覽圖，非原檔）。
- **C（採用）. A＋B，並補上「原檔搬走可重新指定」**——由使用者裁示。

## 決策

### 1. 縮圖持久化（B）

`StoredFileMeta` 新增 `thumb?: string`（`data:` URL）。**這是衍生的小預覽圖，不是原檔**——
ADR-0093「不保存檔案本身」的裁示不變，原檔位元組依然不落地。

政策常數放 `@cinder/engine`（**單一來源，桌面/行動端不會漂移**）：
- `THUMB_MAX_EDGE = 256`（最長邊）、`THUMB_QUALITY = 0.7`（JPEG）、`THUMB_MAX_BYTES = 64KB`
- 超過上限就**不存**（寧可沒縮圖，也不讓儲存膨脹）
- `isThumbnailable(mime)`：只認點陣圖，**排除 SVG**——SVG 是可執行標記（可含 script/外部參照），
  拿去餵 canvas 或當預覽圖都是不必要的攻擊面

縮圖**只存本機**：不進 metadata 訊息、**不上中繼**；收件端自己從位元組產生。

### 2. 讀回原圖（A）

點縮圖 → 燈箱先顯示「能顯示的」（本 session 的原圖 blob，或縮圖），再嘗試從 `savedPath`
**讀回原圖**（Tauri `read_saved_file`）。

### 3. 原檔被搬走 → 重新指定位置

`read_saved_file` 找不到檔案時**不彈任何對話框**（點縮圖不該無預警跳出檔案總管），而是顯示
「找不到原圖」＋**「重新指定位置」**按鈕，由使用者**主動**觸發 `pick_existing_file`。
選定後讀回原圖並**回寫 `savedPath`**——下次就直接讀得到。

瀏覽器沒有任意檔案系統存取 → 明確顯示「此處只能顯示縮圖」，不假裝能讀原檔。

## 理由

- 縮圖解決「**看得見**」（全平台、可靠、跨 session）；`savedPath` 解決「**看原圖**」（零位元組儲存）。
  兩者互補，正好各自補上對方的弱點。
- 「重新指定位置」把 A 的最大弱點（使用者搬檔就永久失效）變成一次點擊即可修復。

## 後果

- 正面：相簿與聊天內嵌縮圖**跨 session 存活**；原圖仍可看；檔案搬走可自行修復。
  ADR-0023 的目標**首次真正達成**（用的是新設計，不是它原本依賴的 B4）。
- 負面 / 已知殘餘風險：
  - **確實多存了縮圖位元組**（每張 ≤64KB，通常 10–30KB）。這是使用者明確裁示接受的取捨；
    桌面上它落在加密 blob 裡（ADR-0054），瀏覽器則是 localStorage 明文——與其他訊息內容同級。
  - 送出端的原圖沒有 `savedPath`（瀏覽器 File API 拿不到完整路徑）→ 自己送出的圖片，
    重載後**只看得到縮圖**，「重新指定位置」仍可手動指回原檔。
  - 縮圖產生需要 canvas：桌面在 `ui/thumbnail.ts`、行動端在 `native/files.ts` 平台縫
    （移植真 RN 改用 expo-image-manipulator）。兩處各約 20 行 canvas 程式碼重複，
    但**政策常數共用**，不會漂移。
- 測試：engine +4（縮圖跨快照存活＝相簿空掉的根因修正、超上限不存、**原檔位元組依然不存**、
  SVG 排除）；desktop +3（重載後只剩縮圖仍渲染為圖片而非灰卡、原圖優先於縮圖、兩者皆無才走檔案卡）。
  全 789 測試通過、typecheck 通過、`cargo check --features tauri-app` 通過。
