# 0114. 行動端功能對齊（一）：修好群訊、補上收回／回應／封鎖

- 狀態：已接受（已實作）
- 日期：2026-07-14
- 相關文件：ADR-0027（群組）、0086（行動端對齊）、0095（群組回條／rumor.id）、
  0100/0101（行動端檔案／通話）、0107（自封副本）、0111（封存）

## 背景與問題

盤點桌面／網頁／行動三者的功能差異。**桌面與網頁是同一份程式碼**（`apps/desktop`，
差異只在 9 個 `isTauri()` 分支，全是平台能力而非功能取捨）。**行動端是另一個 app**，
功能面小得多——而其中有一項是**點了會爆**的。

### 🔴 群訊：行動端點進群組送訊會直接拋錯

```ts
const send = (text: string): void => {
  if (activeId) backendRef.current?.sendMessage(activeId, text); // ← 群組也走這條
};
```

而**群組會出現在行動端的聊天清單裡**（`chatList(contacts, groups, …)`）。

`groupId` 是 **16 bytes hex（32 字元）**，而 pubkey 是 **32 bytes hex（64 字元）**——
把 groupId 丟給 `sendMessage()` 會一路傳到 NIP-44 加密，然後拋出
`Error: second arg must be public key`。

**使用者能點進群組，但一送訊就爆。**

（拋錯其實是幸運的：若 groupId 的長度剛好是 64，它會**靜默**加密給一個不存在的 pubkey
並送出——訊息進了虛空，且沒有任何錯誤訊號。這種失敗模式難察覺得多。）

### 其餘三項缺口（引擎早已支援，只缺 UI 接線）

- **收回訊息**（NIP-09）：**這是隱私問題**。在桌面收回的訊息，行動端會照常顯示；
  而在行動端根本無法收回。ADR-0107 已讓收回**跨自己的裝置同步**——行動端不接就白費了。
- **emoji 回應**（NIP-25）：行動端連**顯示**都不會——對方按的回應完全看不到。
- **封鎖**：安全相關。

## 決策

### 1. 群訊分派（bug fix）

```ts
if (isGroup(activeId)) b?.sendGroupMessage?.(activeId, text);
else b?.sendMessage(activeId, text);
```

並在**引擎**加一條回歸測試釘住這個不變式（`groupId` 不是 pubkey；`sendMessage(groupId)` 必拋錯）
——它屬於引擎契約，不只是行動端的 UI bug。

### 2. 長按＝手機上的「右鍵選單」

行動端沒有 hover，也不該為了回應/收回而塞一排常駐按鈕。**長按訊息**開出操作列：

- 所有訊息：**emoji 回應**（`REACTION_EMOJIS`，與桌面同一組）
- 自己的訊息：另加 **收回**

**已收回的訊息不提供任何操作**（沒有東西可操作了）。

### 3. 已收回的訊息**絕不顯示原文**

顯示「（已收回）」，且**不顯示縮圖、不顯示檔名、不顯示回應、不顯示送出狀態**。
收回的意義就在這裡——半套的收回比沒有收回更糟。已有測試釘住。

### 4. 封鎖：聯絡人長按 → 封鎖；已封鎖區可解除

- **長按才出現**封鎖鈕（避免誤觸把人封鎖掉）——已有測試釘住。
- 封鎖正在檢視的對話 → 自動退回主畫面。
- 封鎖會一併移出聯絡人並清掉該對話（含**封存**，ADR-0111 已處理）。

## 理由

- 群訊那條是**點了會爆**，必須先修。
- 收回是**隱私**功能：ADR-0107 好不容易讓收回跨裝置同步，行動端不接就等於「在手機上收回不了、
  在桌面收回的也留在手機上」——那比沒有收回更危險（使用者以為訊息撤回了）。
- 回應與封鎖的引擎 API 早就存在（`sendReaction` / `blockContact` / `onReaction` / `onBlocked`），
  且後端 `start()` 會回放既有狀態——純 UI 接線，成本低。

## 後果

- 正面：
  - 行動端**能正常收發群訊**。
  - 收回／回應／封鎖上線；`onReaction` / `onUnsend` / `onBlocked` 皆接上後端的開機回放
    → **重載後狀態仍在**。
  - 測試 +8（引擎 1：groupId 不是 pubkey；行動 7：已收回不顯示原文／不顯示狀態／不顯示回應、
    回應渲染、封鎖需長按、已封鎖可解除）。全 repo **910 通過**。

- 行動端**仍缺**（本 ADR 不處理，記錄在案）：
  - **建立群組**與成員管理（目前只能被邀請入群）
  - 通知（Web Notifications）
  - 多身分、本地密碼
  - 貼圖、相簿、AI
  - 上線狀態（只有隱身）、音樂狀態
  - **敲一下只能收、不能發**
  - 多中繼管理、配對搬家（送出端）

- 一個測試教訓：`react-native-web` 產生的 CSS class 含 `r-paddingBlock-…`，
  用字串比對 `"Block"` 會**誤判**。斷言請用 `testID`。
