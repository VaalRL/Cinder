# 0148. 本地暱稱（給聯絡人取只有自己看得到的顯示名，可點切換廣播名）

- 狀態：已接受（已實作）
- 日期：2026-07-16
- 相關文件：**ADR-0061（顯示名稱以加密 profile 廣播）**、0144（改自己的顯示名）、0045/0138（多身分）、
  0071（加密雲端快照）、0125（搬家捆包）、0112/0117（web/mobile nsec 不明文落盤）、0139（自訂對話框）

## 背景與問題

聯絡人的顯示名過去**完全由對方廣播決定**（ADR-0061：對方以加密 kind-0 profile 廣播名字 →
`updateContactName` 無條件覆寫；未收到前退回短 npub）。你無法給聯絡人取一個**只有自己看得到**的名字
（petname／本地暱稱）——像 Signal 的暱稱、通訊錄的備註。使用者要：**自訂暱稱顯示，且點暱稱能切換看
對方廣播的名字**。

關鍵限制：暱稱不能改動 `name`（廣播名獨立保存，供切換/清除還原），否則對方下一次 profile 廣播就會
把暱稱覆寫掉。所以需要一個**獨立欄位**，顯示時恒優先。

## 決策

### 1. 資料：`StoredContact.alias` / `Contact.alias`（本地私有）

`StoredContact` 與 UI 的 `Contact` 各加可選 `alias`。`AppStorage.setContactAlias(pubkey, alias?)`
（空/undefined＝清除）只寫聯絡人區、**不碰 `name`**；memory/local/tauri 三個實作都補上。
`ChatBackend.setContactAlias?` 於 `RelayChatBackend` 實作：寫儲存 → 重載 → 重發聯絡人清單，
**不送任何事件**（純本機）。`BrowserChatBackend`（示範）以記憶體 Map 支援。

- **純本地私有**：暱稱**絕不廣播、絕不送對方或中繼站**。它只隨**你自己的**加密快照（ADR-0071，opt-in）
  與搬家捆包（ADR-0125）在**你自己的裝置間**流動（皆加密、你的金鑰）——與 ADR-0061 廣播自己的名字相反。
- **不覆寫廣播名**：`updateContactName`（ADR-0061）照舊只更新 `name`；`alias` 獨立，故對方改名不影響你的暱稱。

### 2. 顯示：`contactLabel(c) = c.alias?.trim() || c.name`（兩端共用純函式）

engine 匯出 `contactLabel`，各處顯示聯絡人名一律經它（暱稱優先，未設才用廣播名/短 npub）。套用於
桌面側欄/聯絡人列/對話標頭與行動端聊天清單/聯絡人頁/對話標頭；桌面三欄與行動聊天清單的**搜尋/排序**
亦改用暱稱（`buildEntries`／`chatList`）。

### 3. 互動：對話標頭點名字切換、✎ 設定/清除（1:1）

使用者選定「對話標頭點名字」切換（側欄點聯絡人維持開對話，不衝突）：

- **桌面** `ConversationWindow`：有暱稱時標頭名字可點，在**暱稱↔廣播名**間暫態切換（換對話即重置，不落盤）；
  旁邊 ✎（提供 `onSetAlias` 才顯示，即 1:1）以 `prompt`（ADR-0139，預填目前暱稱）設定/清除。
- **行動** `ConversationScreen`：標頭名字可點切換；✎ 展開一列 `TextInput` 設定/清除（留空＝清除）。
- 群組不套用（暱稱是給 1:1 聯絡人的）。

## 後果

- 正面：
  - 可給每位聯絡人取**只有自己看得到**的暱稱，顯示於各處（清單/標頭/搜尋）；**點標頭名字即可切回對方
    廣播的真名**，兩者並存不互相覆寫。
  - 暱稱純本地私有、不外送；只隨你自己的加密快照/搬家在你的裝置間走，符合「明文不上雲、隱私預設」。
  - 顯示邏輯收斂到單一純函式 `contactLabel`，兩端一致；資料層 `alias` 隨既有快照/捆包自動流動，無新協定。
  - 測試 +8（engine storage setContactAlias／relay-backend 不外送＋不被廣播覆寫＋清除；desktop
    deck-sidebar 暱稱優先＋ConversationWindow 標頭 3；mobile 標頭 2）。全綠 engine 243／desktop 333／
    mobile 145／i18n 8。

- 已知限制／取捨：
  - 暱稱同步依附**你自己的**加密快照（ADR-0071，需 opt-in）與搬家捆包；未開快照＝純單機（換裝置要重設）。
  - 標頭切換是暫態（不記住哪個對話正顯示廣播名）；換對話即回暱稱優先——簡單、可預期。
  - 暱稱只給 1:1 聯絡人；群組成員名、訊息請求區仍走廣播名/短 npub。
  - 大小寫敏感、僅 `trim()`（同 ADR-0146 風格）。
