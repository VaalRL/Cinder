# 0154. 頭像加密廣播給聯絡人——內嵌縮圖；網址只當輸入方式

- 狀態：已接受（已實作）
- 日期：2026-07-16
- 相關文件：ADR-0061（加密個人檔廣播——本 ADR 擴充其 content）、0077（本地頭像——顯示
  優先序的本地覆寫層）、0071（雲端快照——本 ADR 明確剝除 avatar）、0002（隱私基線）

## 背景與問題

ADR-0077 的頭像是純本地的：只有自己看得到，聯絡人看到的仍是生成頭像。使用者要
「對方也看得到我的頭像」，且頭像來源要有兩種：上傳檔案、或填寫圖片網址。

Nostr 一般做法（kind 0 公開個人檔放 `picture` URL）有兩個 Cinder 不能接受的問題：

1. **公開**：名字↔npub 對映與頭像公開給全世界（ADR-0061 已否決過）。
2. **URL 當傳輸＝追蹤信標**：把網址發給聯絡人、由**對方裝置**去抓，等於讓每個聯絡人
   把自己的 IP、上線時間送給頭像主人指定的任意主機——Gift Wrap 藏起來的社交圖譜被一
   次 HTTP GET 掀開。內容還可事後調包、斷網失效、可指向內網。

## 決策

**頭像以內嵌縮圖走 ADR-0061 的加密個人檔廣播；「網址」只是輸入方式，不是傳輸方式。**

1. **線上格式**：`wrapProfile` 的 rumor content 從 `{name}` 擴成 `{name, avatar?}`。
   `avatar` 為 data URI 縮圖（來源端 128px、JPEG 壓縮，實測 3–8KB）；`""` 為「已移除」
   記號（收端清掉既存頭像）；**欄位缺席＝無變更**（不清不改）。舊版客戶端只讀 `name`，
   多的欄位自動忽略——線上格式向後相容，零協定版本切換。
2. **收端防禦**：`parseProfile` 只接受 `data:image/(jpeg|png|webp|gif);base64,` 前綴且
   ≤ `PROFILE_AVATAR_MAX_BYTES`（48KB）的頭像；超大或非點陣圖（含 SVG——可執行標記，
   同 ADR-0102 的排除理由）一律丟棄。收到的頭像存 `StoredContact.avatar`，隨聯絡人
   持久化；對「請求區」的人也學（同 ADR-0121 名稱的理由——請求列表要認得出人）。
3. **網址＝輸入**：使用者填網址時，由**本人裝置** fetch → canvas 縮圖 → 內嵌廣播。
   聯絡人永遠只收密文縮圖，不接觸網址。已知限制：瀏覽器/webview 的 fetch 受 CORS
   管制，不允許跨站的圖床會失敗（明確報錯，請使用者下載後改用上傳）；未來可用
   tauri HTTP plugin 繞過（桌面原生 fetch），另立 ADR。
4. **雲端快照剝除 avatar**：`buildSnapshotContent` 夾帶整份 `StoredContact[]`，
   30 個聯絡人的頭像即撐爆 180KB 明文預算（relay 單顆 256KB 上限）→ 組裝時剝掉
   `avatar` 欄位（contacts 與 blocked 皆然）。換機後由開機廣播重新學到，不損失。
   搬家捆包（ADR-0072）不剝——它本來就載整包訊息歷史，頭像是零頭，且能讓新裝置
   立刻有頭像。
5. **隱私不倒退**：既有 ADR-0077 本地頭像**不會**自動升級成廣播。廣播只由新的
   `setSelfAvatar` 明確觸發（UI 選單標示「會同步給聯絡人」）；自身廣播頭像另存
   （`selfAvatar`，隨 `StorageSnapshot` 持久化），與 0077 的本地覆寫層無關。移除頭像
   持久化為 `""` 並繼續隨開機廣播送出——確保晚上線的聯絡人也會清掉舊圖。
6. **顯示優先序**：本地覆寫（0077，`nb.avatar.<pk>`）＞ 對方廣播（`Contact.avatar`）
   ＞ 生成頭像。與 ADR-0148 本地暱稱恒優先於廣播名同一哲學。
7. **變更即重播**：`setSelfAvatar` 比照 `setSelfName`（ADR-0144）清空 `profileSentTo`
   後全量重播。開機廣播照舊（頭像跟著名字走）；「僅變更時送」的節流留作未來優化
   （同 0061 既記載的方向）。

## 三平台入口（本 ADR 一併補齊）

- 桌面/瀏覽器（同一套 React）：經典佈局「我」區與對話視窗原有入口照舊；**補上三欄
  佈局側欄（DeckSidebar）自己頭像可編輯**；選單新增「從網址設定…」。
- 行動端：新增設定頁「頭像」區（更換／從網址／移除＋預覽）；聊天清單與聯絡人清單
  顯示廣播頭像（原本只有生成色圓）。

## 理由

- 隱私零新增面：與名稱同級——密文、藏社交圖譜、明文不上雲全部不變。
- 流量可控：換一次頭像＝聯絡人數 × ~30KB（兩層加密後）；中繼只暫存 ≤7 天、
  每收件人 500 格中占 1 格。
- Fix First：完全重用 0061 的 wrap/parse/廣播/互惠管線，只擴 content 欄位。

## 後果

- 正面：聯絡人看得到你自選的頭像；三平台都有入口；離線可用（縮圖內嵌）。
- 負面／已知限制：
  - 開機廣播因頭像變重（每聯絡人 ~30KB）；未來可做 hash 比對「僅變更時送」。
  - 群組內非聯絡人成員看不到你的頭像（廣播逐聯絡人）——設計如此。
  - 自身頭像不隨雲端快照同步（快照剝除 avatar 的直接結果；搬家捆包會帶）。
  - 網址輸入受 CORS 限制（見決策 3）。
- 測試：core（wrap/parse round-trip、移除記號、超大/壞 mime 丟棄）＋engine（儲存、
  收發、快照剝除、變更重播）＋desktop（優先序、選單、DeckSidebar 入口）＋mobile
  （設定區、清單顯示）。
