# 0159. 下班時間發訊提示——發訊端非阻斷橫幅（ADR-0157 的另一半）

- 狀態：已接受（已實作）
- 日期：2026-07-16
- 相關文件：ADR-0157（表定上下班＋收訊端下班靜音——本 ADR 補發訊端的資訊對稱）

## 背景與問題

ADR-0157 的下班靜音只作用在收訊端：發訊者無感——下班時間傳訊給同事，對方不會被
通知，但發訊者不知道，容易把「沒被提醒」誤讀成「已讀不回」。使用者裁示：輸入框
顯示非上班時間提示；**訊息仍照常送達，只是沒有通知**（後者為 0157 既有行為，本 ADR
明文確認，不改）。

## 決策

1. **非阻斷提示橫幅**：對話對象是**組織成員（1:1）或組織群組**、且當下在表定時間外
   → 對話視窗輸入區上方顯示一條淡色橫幅：「目前非上班時間（HH:MM–HH:MM）——對方的
   通知已靜音，可能不會即時看到；訊息照常送達。」**不擋送出**（緊急事務照傳）。
2. **同一份判定資料**：班表與成員名單隨名冊分發、兩端一致（`orgInfo`）——發訊端以
   `inWorkHours`（ADR-0157 核心純函式）自行判定，無需任何協定往返。
3. **即時性**：視窗開著跨越上下班邊界時，每分鐘重算一次（僅組織對話啟用計時器）；
   上班時間自動消失；非組織對話完全不渲染。
4. **明文確認 0157 語意**：下班訊息**照常加密送達、照常入庫、未讀照常累計**——
   靜音只作用於通知彈窗與音效。本 ADR 不觸碰投遞路徑。

## 後果

- 正面：發訊端與收訊端資訊對稱——「為什麼對方沒回」有了正確的預期；零協定變更。
- 已知限制：以發訊者本地時鐘判定（同 0157，無時區/多班表）；SSR 測試以 `nowMinutes`
  測試 prop 注入時間（同 `initialSoundEditing` 慣例）。行動端未接（同 0157）。
- 測試：desktop（下班顯示/上班隱藏/非組織對話不渲染）。
