# 0163. 入職金鑰託管＋離職一鍵接管——公司帳號模型（揭露、共識、可執行）

- 狀態：已接受（已實作）
- 日期：2026-07-16
- 相關文件：ADR-0156（入職邀請/自動核准——託管走同一管線）、0047/0052（名冊移除/輪替＝
  離職語意）、0044（relay allowlist——離職的**真正**執行點）、0112（靜態加密）、
  0002（隱私鐵則——本 ADR 以「揭露的公司資產」明確劃出企業例外的邊界）

## 背景與問題

企業要求：離職時企業主能接管員工身份、封鎖其登入、查看/刪除。研究（前一輪）結論：
**「離職時才抽金鑰」在密碼學上不可行**（金鑰在員工裝置、無法遠端沒收；不合作的離職者
恰好防不住），且會讓在職員工天天暴露於「一個標記即被接管」的單點風險。可行且合理的
形態是**時間點前移到入職**：公司帳號＝雇主持有金鑰備份（等同公司信箱的 IT 管理權），
**揭露且經員工同意**；離職走名冊/allowlist（可執行的切斷），接管用入職時的託管金鑰。

## 決策

### 1. 入職金鑰託管（揭露、opt-in 於邀請碼）

- 邀請碼加旗標 `OrgInvite.escrow`：企業主產生邀請碼時勾選「公司帳號（雇主持有金鑰備份）」。
  員工端貼碼時 UI **明示**此性質（登入/新增身分面板皆顯示托管警語），持久化
  `Profile.orgEscrow`。
- **傳輸即加密**：入職請求（ADR-0156）本就是 Gift Wrap E2E 給管理者——`wrapOrgJoin` 擴帶
  `nsec`，等同「以管理者公鑰加密員工私鑰」，中繼只見密文，管理者離線也收得到。
  員工端僅在 `orgEscrow` 時附上 nsec。
- **管理者端持久化**：引擎驗權杖核准後發 `onOrgEscrow(member, name, nsec, relayUrl)`；App 存
  `nb.orgEscrow.<admin>`（隨管理者身分的加密儲存落地，ADR-0112）。

### 2. 離職（名冊移除＝可執行的封鎖）

企業主在名冊管理移除該成員（既有能力，ADR-0047/0052）→ 重新簽發名冊：
- 全員通訊錄自動移除、組織群組踢出；
- `rosterAllowlist` 不再含其 pubkey → 佈建到公司 relay 的 `allowedAuthors` 後，
  **舊金鑰連公司站都發不出任何事件**——這是「無法登入」在組織語境內**真正可執行**的形態。

### 3. 離職接管（用託管金鑰，在管理者裝置上）

設定 →「離職帳號」區列出**已託管但不在現行名冊中**（＝已離職）的條目，每筆兩個動作：
- **接管登入**：以託管 nsec 匯入為本機新身分（`Profile.orgOffboarded: true`，名稱
  `離職·<name>`），沿用既有 `addIdentity` 匯入路徑；切換過去即可連 relay 收取該身分
  離線信箱殘留、查看/導出，之後以既有身分移除刪掉。
- **刪除**：移除該託管條目。

### 4. 誠實的極限（必須寫入，不得隱含承諾）

- **接管看到的「歷史」＝relay 仍持有的離線信箱殘留**（保留期內、ADR-0160 可延長）＋
  接管後續收到的；**員工裝置本機的聊天記錄無法遠端取得**（那從不上雲）。組織對話的
  完整副本其實在**同事端**——離職不會讓組織歷史消失。
- 託管防得住合作的員工的裝置遺失/交接；**防不住**在職期間已自行備份金鑰、截圖、外流的
  員工——這是任何通訊系統（含公司信箱）的物理極限，UI 與 ADR 不假裝能防。
- 未勾 escrow 的邀請＝無託管（一般工作身分）；既有名冊/allowlist 封鎖仍適用。

## 理由

- 揭露＋入職同意把「監控」轉為「公司資產管理」——業界標準（公司信箱/MDM）的正當模型。
- 復用既有管線：託管走 0156 入職 Gift Wrap、封鎖走 0044 allowlist、接管走既有身分匯入
  ——零新協定面、零新加密原語（Fix First）。

## 後果

- 正面：離職接管/封鎖成為一鍵流程且**技術上真的執行得到**（allowlist）；在職員工的金鑰
  暴露僅發生在**入職那一次、且經揭露同意**，不是天天可被單方接管。
- 已知限制：見決策 4；接管需管理者裝置有託管條目（換管理者裝置需帶著加密儲存/搬家）；
  行動端未接（同既有企業功能）。
- 測試：core（invite escrow round-trip、wrapOrgJoin 帶/不帶 nsec、parseOrgJoin）＋engine
  （orgEscrow 時入職帶 nsec、管理者收 onOrgEscrow、未 escrow 不帶）＋desktop（託管持久化
  純函式、離職接管清單分流、入職揭露文案）。
